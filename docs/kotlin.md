# Intro

Why Kotlin?

statically Typed Multiparadigm language Designed as Better alternative to Java

- concise syntax
- type inference
- functional patterns
- Null Safety
- more

2016 Stable release

named after Kotlin Island in Russia
can interopt with Existing java Code
runs on JVM.

Old java project can be migrated step by step to kotlin

In addition to JVM, Kotlin can compile to

- Javascript

Opening the door to multiplattform apps:

- preferred Langauge for Android App development

On of the Killer features:

- Coroutine
- simplified way to write asynchronous non blocking code
  - common requirement mobile dev

## Intro coding

- Type inferrence automatically or also Explicit
- cannot be Null unless you allow it with `var hello: String?`
- simicolons are optional -> Line breaks are significant
- can OOP patterns but also Functional Pattern
  - Special functional things:
    - modify the behaviour of a class without `inheritance`using `extensions`

- functions are first class objects can be stored as variables, passed as arguments or used anonimously with lambdas
- allows destructuring
